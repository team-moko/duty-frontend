# Handoff: 절세 서비스 — 서비스 진입 랜딩 페이지

## Overview
절세(節稅) 추천 서비스의 **첫 진입 랜딩 페이지(신규)** 를 정의합니다. STEP 1 입력 플로우 앞단에 놓여, 서비스의 **간편함**을 강조하고 **"내 절세 전략 찾기"** 버튼으로 STEP 1로 유입시키는 게 목표입니다.

핵심 목표: 처음 방문한 사용자가 "복잡할 것 같다"는 진입 장벽 없이, 30초·무가입·무료라는 점을 즉시 이해하고 곧바로 진단을 시작하게 만드는 것.

---

## About the Design Files
이 번들의 파일들은 **HTML/React(JSX)로 만든 디자인 레퍼런스**입니다 — 의도한 룩앤필과 동작을 보여주는 프로토타입이며, 그대로 복사해 프로덕션에 넣을 코드가 아닙니다.

작업은 이 디자인을 **타겟 코드베이스의 기존 환경(React/Vue/Next 등)과 패턴·컴포넌트·디자인 토큰을 사용해 재현**하는 것입니다. 아직 프론트엔드 환경이 없다면 프로젝트에 가장 적합한 프레임워크를 선택해 구현하세요.

`.jsx` 파일들은 순수 데모 목적이라 inline style + 전역 `window` 등록 방식으로 작성돼 있습니다. 실제 구현에서는 코드베이스의 스타일링 방식(CSS Modules, styled-components, Tailwind 등)으로 옮기세요.

## Fidelity
**High-fidelity (hifi).** 최종 색상·타이포그래피·간격이 확정된 목업입니다. 아래 디자인 토큰과 스펙을 그대로 사용해 픽셀 단위로 재현하세요. 폰트는 **Pretendard Variable**.

## Screen / View

### 서비스 진입 랜딩 페이지 (신규)
- **파일**: `design_files/screen-landing.jsx` (컴포넌트 `ScreenLanding`)
- **URL 예시**: `jeolse.kr/` (서비스 루트)
- **목적**: 처음 방문자에게 서비스의 간편함(30초·무가입·무료)을 각인시키고 STEP 1으로 유입.
- **디자인 방향**: 앱 내부 입력 화면과 톤을 맞추되, 히어로에서 소프트 블루 그라데이션으로 진입 분위기를 만들고 흰 바탕에 정보를 순차 배치.

**레이아웃 (위→아래, 모바일 width 390px 기준):**

1. **앱바**: 좌측 정렬 "절세" 로고 타이틀(19px/800). 뒤로가기 없음(`home`), 투명 배경(`bare`).

2. **히어로** — 배경 `radial-gradient(130% 70% at 50% -6%, #EAF1FE 0%, #F4F8FF 40%, #fff 78%)`, 패딩 `20px 24px 40px`:
   - **배지**: 흰 pill + `1px solid #E5E8EB` + `TT.card` 그림자, 좌측 7px 초록 점(`#12B886`) + "가입 없이 · 무료로"(13px/700, `#1D52C9`)
   - **헤드라인**: "몰라서 못 받던 세금, / **30초**면 찾아드려요"(32px/800, `#191F28`, line-height 1.28, letter-spacing -0.035em). "30초"만 파랑(`#2F6BF0`)
   - **서브카피**: "복잡한 절세, 이제 계산기 두드리지 마세요. 몇 가지만 고르면 내게 가장 유리한 전략을 바로 알려드려요."(16px/500, `#4E5968`, line-height 1.6)
   - **미리보기 스탯 카드**: 흰 카드(radius 20, `TT.cardHi` 그림자), 좌우 2분할(가운데 `1px` 세로 구분선):
     - 좌: 캡션 "예상 추가 환급액"(12.5px/600, `#8B95A1`) + 값 "+84만원"(27px/800, "만원"은 17px)
     - 우: 캡션 "추천 계좌 조합" + 파란 칩 2개(연금저축 / IRP, `AccountChip tone="blue"`)
     - ⚠️ 이 수치(+84만원)는 **예시용 플레이스홀더**입니다. 개인 결과가 아니며, 실제로는 대표 예시값 또는 애니메이션 롤링 등으로 처리 정책 결정 필요.

3. **특징 3줄** — 패딩 `10px 24px`, 행 간 `gap 20`. 각 행: 46px 아이콘칩(`#EAF1FE` 배경, radius 14, 파란 인라인 SVG) + 제목(17px/700) + 설명(14px/500, `#4E5968`):
   - **30초면 충분해요** — "나이·연봉 몇 가지만 고르면 끝. 복잡한 서류도, 로그인도 필요 없어요." (시계 아이콘)
   - **내 조건에 딱 맞게** — "내 소득 구간에서 가장 유리한 절세 계좌 조합만 골라서 추천해요." (과녁 아이콘)
   - **환급액까지 숫자로** — "얼마를 더 돌려받을 수 있는지 예상 금액과 환급률로 바로 보여드려요." (막대그래프 아이콘)

4. **진행 방식 3스텝** — `#F7F8FA` 카드(radius 20). 라벨 "이렇게 진행돼요"(13px/700, `#8B95A1`). 각 스텝: 30px 파란 원형 번호(01/02/03) + 세로 연결선(`#E5E8EB`) + 제목(15.5px/700) + 설명(13.5px/500, `#8B95A1`):
   - **01 기본 정보 입력** — 나이·연봉·투자 현황
   - **02 맞춤 분석** — 유리한 계좌 조합 계산
   - **03 전략 확인** — 예상 환급액·추천 순위

5. **신뢰 라인** (가운데 정렬): "입력한 정보는 저장되지 않아요. 계산에만 쓰이고 바로 사라집니다."(13px/500, `#8B95A1`)

6. **고정 CTA**: `position: sticky; bottom: 0`, 위쪽으로 흰색 그라데이션 페이드. 파란 `CTAButton`, 메인 "내 절세 전략 찾기" + 서브 "30초면 끝나요 · 가입 불필요".

## Interactions & Behavior
- **"내 절세 전략 찾기" CTA**: 탭 시 STEP 1 입력 플로우(`/step/1`, 기본 정보) 시작. 가입/로그인 없이 진입 가능해야 함 (핵심 진입점).
- **스크롤**: 단일 세로 스크롤 원페이지. CTA는 스크롤 내내 하단 고정.
- 반응형: 모바일 우선(디자인 기준 width 390px). 데스크톱에서는 콘텐츠를 중앙 정렬된 최대폭 컬럼으로.

## Content Notes
- **미리보기 스탯(+84만원, 연금저축/IRP)** 은 실제 사용자 데이터가 아닌 **대표 예시**입니다. 개인화된 값처럼 오해되지 않도록 카피/처리 정책을 정하세요(예시값 고정 표기, "예시" 라벨, 또는 여러 케이스 롤링 등).
- **간편함 3요소(30초·무가입·무료)** 가 이 페이지의 핵심 메시지 축입니다. 배지·헤드라인·CTA 서브카피 세 군데에서 반복 강조됩니다 — 카피 변경 시 이 일관성을 유지하세요.
- 정보 미저장 문구는 실제 데이터 처리 정책과 일치해야 합니다(문구만 넣지 말 것).

## Design Tokens
`design_files/theme.jsx`의 `TT` 객체가 원본. 주요 값:
- **Brand**: blue `#2F6BF0`, blueDeep `#1D52C9`, blueWeak `#EAF1FE`, blueWeak2 `#F4F8FF`
- **Green**(강조 점): green `#12B886`
- **Neutrals**: ink `#191F28`, ink2 `#4E5968`, ink3 `#8B95A1`, ink4 `#B0B8C1`, line `#E5E8EB`, line2 `#F2F4F6`, fieldBg `#F7F8FA`, appBg `#F2F4F6`, white `#FFFFFF`
- **Type**: Pretendard Variable (fallback: Pretendard, -apple-system, system-ui, sans-serif)
- **Radius**: 아이콘칩 14, 스탯/스텝 카드 20, 버튼 16, pill 999
- **Shadow**: card = `0 1px 2px rgba(25,31,40,0.04), 0 6px 20px rgba(25,31,40,0.05)`; cardHi(스탯 카드) = `0 2px 4px rgba(25,31,40,0.05), 0 12px 32px rgba(47,107,240,0.12)`; CTA = `0 6px 18px rgba(47,107,240,0.28)`

## Assets
- **폰트**: Pretendard Variable — CDN `https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/variable/pretendardvariable.min.css`. 코드베이스에 이미 폰트 시스템이 있으면 그것을 사용.
- **아이콘**: 모두 인라인 SVG(시계·과녁·막대그래프·초록 점 등). 코드베이스의 아이콘 라이브러리로 대체 가능.
- 이모지·별도 비트맵 이미지 없음.

## Files

- `design_files/theme.jsx` — 디자인 토큰 + 공용 UI 프리미티브(AppBar, CTAButton, AccountChip 등)
- `design_files/screen-input-stepped.jsx`, `screen-input2.jsx` — CTA가 연결되는 STEP 1 입력 플로우(참고용)

- `design_files/design-canvas.jsx` — 프로토타입 실행에 필요한 의존 파일(참고용)
