# Handoff: 절세 서비스 — 공유 플로우 & 유형 시스템

## Overview
절세(節稅) 추천 서비스의 **공유(share) 기능**과 관련한 4가지 작업을 정의합니다.

1. **STEP2 추천 결과 페이지** — 하단 공유 바 제거
2. **조합 상세 페이지** — 하단 CTA를 "공유하기" 버튼 하나로 교체
3. **공유 링크 랜딩 페이지 (신규)** — 공유받은 사람이 보는 원페이지. 서비스 유입이 목표
4. **절세 유형 시스템 (신규)** — 예상 환급률(%)로 나뉘는 심리테스트식 "유형" 4종

핵심 목표: 사용자가 자신의 추천 전략을 공유 → 링크를 받은 지인이 랜딩 페이지에서 흥미를 느끼고 **"내 절세 전략 보러가기"** 버튼으로 서비스에 신규 유입되는 바이럴 루프.

## About the Design Files
이 번들의 파일들은 **HTML/React(JSX)로 만든 디자인 레퍼런스**입니다 — 의도한 룩앤필과 동작을 보여주는 프로토타입이며, 그대로 복사해 프로덕션에 넣을 코드가 아닙니다.

작업은 이 디자인을 **타겟 코드베이스의 기존 환경(React/Vue/Next 등)과 패턴·컴포넌트·디자인 토큰을 사용해 재현**하는 것입니다. 아직 프론트엔드 환경이 없다면 프로젝트에 가장 적합한 프레임워크를 선택해 구현하세요.

`.jsx` 파일들은 순수 데모 목적이라 inline style + 전역 `window` 등록 방식으로 작성돼 있습니다. 실제 구현에서는 코드베이스의 스타일링 방식(CSS Modules, styled-components, Tailwind 등)으로 옮기세요.

## Fidelity
**High-fidelity (hifi).** 최종 색상·타이포그래피·간격·인터랙션이 확정된 목업입니다. 아래 디자인 토큰과 스펙을 그대로 사용해 픽셀 단위로 재현하세요. 폰트는 **Pretendard Variable**.

## Screens / Views

### 1. STEP2 추천 결과 페이지 (수정)
- **파일**: `design_files/screen-result.jsx`
- **변경 내용**: 화면 하단에 고정(sticky)돼 있던 **공유 바 전체를 제거**했습니다.
  - 제거된 것: 좌측 공유 아이콘 버튼(56×56) + 우측 파란 CTA "내 절세 전략 공유하기"
- **이유**: 결과 목록 페이지에서는 공유를 노출하지 않고, 개별 조합 **상세 페이지에 들어갔을 때만** 공유하도록 정리.
- **레이아웃 영향**: 조합 리스트 컨테이너의 하단 패딩을 `140px → 40px`로 축소(고정 바가 사라졌으므로).
- 이 페이지에는 이제 하단 고정 버튼/바가 **없습니다**. 조합 카드를 탭하면 상세 페이지로 이동.

### 2. 조합 상세 페이지 (수정)
- **파일**: `design_files/screen-detail.jsx`
- **변경 내용**: 하단 고정 영역을 **"공유하기" 버튼 하나**로 교체.
  - 제거: 기존의 공유 아이콘 버튼 + "이 전략으로 시작하기" 파란 CTA
  - 신규: 전체폭(width 100%) 흰색 버튼, 높이 56px, radius 16px, `1px solid #E5E8EB` 테두리, `TT.card` 그림자. 내부에 공유 아이콘(share-nodes 스타일 SVG) + 텍스트 "공유하기"를 `gap:8`로 가로 배치. 텍스트 `16px / 800 / color #191F28 / letter-spacing -0.02em`.
- **위치**: 화면 하단 `position: sticky; bottom: 0`, 패딩 `14px 20px 30px`, 위쪽으로 앱 배경색 그라데이션 페이드.

### 3. 공유 링크 랜딩 페이지 (신규) — 핵심
- **파일**: `design_files/screen-share.jsx` (컴포넌트 `ScreenShare`)
- **URL 예시**: `jeolse.kr/s/{공유자이름}`
- **목적**: 링크를 받은 사람이 공유자의 추천 전략을 매력적으로 보고, 흥미가 생겨 자기 전략을 확인하러 서비스에 유입.
- **개인정보 노출 금지**: 공유자의 **환급 금액(원), 나이, 연봉은 절대 표시하지 않습니다.** 노출 가능한 것은 예상 환급률(%), 추천 계좌 조합, 절세 유형뿐.
- **디자인 방향**: 앱 내부 화면(블루 히어로 + 흰 시트) 틀에서 벗어나, 공유받은 **"카드/티켓"** 느낌의 독립 레이아웃.

**레이아웃 (위→아래, 모바일 width 390px 기준):**
1. **배경**: 상단에서 퍼지는 소프트 블루 방사형 그라데이션 — `radial-gradient(120% 60% at 50% 0%, #EAF1FE 0%, #F2F4F6 46%)`
2. **발신 라인**: 좌측 30px 원형 아바타(공유자 이니셜, 파란 그라데이션 배경) + "{이름}님이 절세 전략을 공유했어요" (이름은 800 굵게)
3. **공유 카드** (흰색, radius 24, 그림자 `0 12px 40px rgba(29,82,201,0.14), 0 2px 6px rgba(25,31,40,0.05)`):
   - **상단 = 절세 유형 결과** (아래 §4 참조). 배경 `linear-gradient(180deg, #F4F8FF 0%, #fff 100%)`
   - **티켓 절취선**: 좌우 끝에 배경색과 같은 16px 반원 노치 2개 + 그 사이 `2px dashed #E5E8EB` 가로선
   - **하단 = 지표**: 가운데 정렬. "계좌를 잘 조합하면" → 큰 숫자 `16.5%`(72px/800, 파랑 `#2F6BF0`) → "까지 돌려받을 수 있어요" → 캡션 "납입액 대비 예상 환급률 기준"(12.5px, `#B0B8C1`)
4. **추천받은 계좌 조합**: 라벨 "추천받은 계좌 조합"(13px/700, `#8B95A1`) + 칩 3개(연금저축 / IRP / 청년형 ISA) — **세 칩 모두 동일한 파란 톤**(`AccountChip tone="blue"`). ⚠️ 첫 칩만 다른 색으로 두지 말 것(선택된 것처럼 오해됨). 아래 인사이트 박스: `#F4F8FF` 배경, radius 12, 💡 + "세액공제 16.5% 구간 + 청년 비과세 혜택까지 모두 챙기는 조합이에요"
5. **전환 훅** (가운데 정렬): "그럼 **나는** 얼마나 돌려받을 수 있을까요?"(23px/800, "나는"만 파랑) + 서브 "사람마다 유리한 전략이 달라요. 내 조건으로 30초 만에 확인해보세요."(14.5px/500, `#8B95A1`)
6. **혜택 3줄** (흰 카드, 각 행 사이 `1px solid #F2F4F6` 구분선): 42px 아이콘칩(`#F7F8FA` 배경) + 제목(15.5px/800) + 설명(13px/500)
   - ⏱️ **30초면 끝** — "나이·연봉만 입력하면 바로 결과가 나와요"
   - 🎯 **내 조건에 딱 맞게** — "내 소득 구간에 유리한 계좌 조합만 추천"
   - 💸 **연 환급액까지 계산** — "얼마를 돌려받을 수 있는지 금액으로 확인"
   - 카드 아래 캡션: "가입·로그인 없이 바로 결과를 볼 수 있어요"(12px, `#B0B8C1`)
7. **고정 CTA**: `position: sticky; bottom: 0`. 파란 CTAButton, 메인 "내 절세 전략 보러가기" + 서브 "30초면 충분해요". 클릭 시 STEP1 입력 페이지로 이동(유입 진입점).

### 4. 절세 유형 시스템 (신규)
- **파일**: `design_files/screen-share.jsx` — 데이터 `TAX_TYPES`, 선택 함수 `getTaxType(rate)`, 단독 카드 컴포넌트 `ShareTypeCard`.
- **분류 기준**: **예상 환급률(%) 단 하나**로 나눕니다. 서비스에서 정량화된 유일한 축이기 때문.
- **프레이밍 원칙 (중요)**: 낮은 %도 부정적으로 두지 않습니다. 낮을수록 "이미 잘하고 있는 고수", 높을수록 "아직 챙길 게 많은 기회" — **모든 유형이 기분 좋게** 느껴져야 양쪽 다 유입됩니다. (낮은 사람=뿌듯함으로 확인, 높은 사람=기회감으로 행동)
- **4개 구간** (`rate >= min` 순으로 첫 매치):

  | 구간(%) | 이모지 | 유형명 | 태그라인 | 키워드 |
  |---|---|---|---|---|
  | ≥ 15 | 🚀 | 환급 떡상 잠재력형 | "아직 안 챙긴 혜택이 이렇게나! 지금이 기회예요" | #숨은환급_발굴 #최대치도전 #지금이기회 |
  | 13–15 | 🧩 | 빈틈없는 균형형 | "받을 수 있는 혜택은 하나도 안 놓쳐요" | #세액공제_맥스 #비과세_알뜰 #장기전략가 |
  | 11–13 | 🌱 | 알뜰 실속형 | "굴리는 맛까지 챙기면서 기본까지 딱" | #유동성_중시 #실속파 #균형감각 |
  | < 11 | 🏆 | 이미 절세 고수형 | "웬만한 혜택은 이미 챙기고 계셔요" | #공제_거의만렙 #빈틈없음 #절세_마스터 |

- **유형 카드 (`ShareTypeCard`) 스펙** (첨부 유형별 4개 아트보드 참고):
  - 컨테이너: radius 24, 배경 `linear-gradient(180deg, #F4F8FF 0%, #fff 100%)`, 패딩 `30px 26px 34px`, 가운데 정렬, 그림자 `0 12px 40px rgba(29,82,201,0.14), ...`
  - 상단 배지: 파란 배경(`#2F6BF0`) 흰 텍스트 pill, "{이름}님의 절세 유형"(13px/800)
  - 이모지: 62px
  - 유형명: 28px / 800 / `#191F28` / letter-spacing -0.035em
  - 태그라인: 15px / 600 / `#4E5968`, 좌우 큰따옴표(“ ”)로 감쌈
  - 키워드 칩: 흰 배경 + `1px solid #E5E8EB` 테두리, radius 999, `6px 12px`, 12.5px / 700 / `#4E5968`, `gap 7`로 가운데 wrap

## Interactions & Behavior
- **결과 페이지 → 상세**: 조합 카드 탭 시 해당 조합 상세 페이지로 네비게이션.
- **상세 "공유하기" 버튼**: 탭 시 OS 공유 시트(Web Share API `navigator.share`) 또는 카카오 공유 등 트리거. 공유되는 URL은 `jeolse.kr/s/{식별자}` 형태의 랜딩 링크.
- **공유 랜딩 "내 절세 전략 보러가기"**: 탭 시 STEP1 입력 플로우 시작(신규 방문자 유입 진입점). 가입/로그인 없이 진입 가능해야 함.
- **유형 결정**: 공유 시점의 예상 환급률(%)을 `getTaxType(rate)`에 넣어 유형을 확정. 랜딩 페이지는 공유된 rate/유형/조합을 링크 파라미터 또는 서버 조회로 렌더.
- 반응형: 모바일 우선(디자인 기준 width 390px). 데스크톱에서는 카드를 중앙 정렬된 최대폭 컬럼으로.

## State Management
- 공유 랜딩 페이지가 받아야 하는 데이터: `{ 공유자표시이름, rate(%), 추천조합 배열, (rate로부터 파생되는) 유형 }`.
- 유형은 저장하지 않고 rate에서 파생해도 됨(`getTaxType`). 단, 유형 정의가 바뀔 때 과거 공유 링크의 표시가 달라질 수 있으니 정책 결정 필요.
- **개인정보(금액/나이/연봉)는 공유 페이로드에 포함하지 말 것.**

- **Green**(효과 강조용): green `#12B886`, greenDeep `#0CA678`, greenWeak `#E6F8F1`
- **Neutrals**: ink `#191F28`, ink2 `#4E5968`, ink3 `#8B95A1`, ink4 `#B0B8C1`, line `#E5E8EB`, line2 `#F2F4F6`, fieldBg `#F7F8FA`, appBg `#F2F4F6`, white `#FFFFFF`
- **Type**: Pretendard Variable (fallback: Pretendard, -apple-system, system-ui, sans-serif)
- **Radius**: 칩 8, 인사이트박스 12, 버튼 14~16, 카드 20~24, pill 999
- **Shadow**: card = `0 1px 2px rgba(25,31,40,0.04), 0 6px 20px rgba(25,31,40,0.05)`; 공유카드 = `0 12px 40px rgba(29,82,201,0.14), 0 2px 6px rgba(25,31,40,0.05)`; CTA = `0 6px 18px rgba(47,107,240,0.28)`

## Assets
- **폰트**: Pretendard Variable — CDN `https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/variable/pretendardvariable.min.css`. 코드베이스에 이미 폰트 시스템이 있으면 그것을 사용.

- **이모지**: 유형/혜택의 이모지(🚀🧩🌱🏆⏱️🎯💸)는 시스템 이모지 그대로 사용.

## Files
